# Emotion-Mechanic Pairs (Extended)

## Social Anxiety
- **Mechanic:** Assemble fragmented sentences from floating word pieces while a timer ticks audibly.
- **Variation:** Other characters’ faces drift closer as the timer runs low, encroaching on screen space.
- **Twist:** Correct words sometimes fall apart after placement, forcing reassembly (unpredictable self-sabotage).

## Monotony / Burnout
- **Mechanic:** Perform the exact same 3-input sequence for 30 seconds. Each cycle the screen desaturates slightly.
- **Variation:** Rarely, a different input appears. If taken, the sequence breaks and must restart—temptation of change within routine.
- **Twist:** After enough cycles, the character begins auto-completing the sequence without player input, showing learned helplessness.

## Grief / Numbness
- **Mechanic:** Walk toward a glowing memory. Movement speed is halved and each step requires a deliberate button press.
- **Variation:** The memory splits into fragments scattered across the level. Collecting all is possible but exhausting.
- **Twist:** Some fragments fade when approached, leaving empty space. The player learns to accept absence.

## Cognitive Overload
- **Mechanic:** Three concurrent mini-games on screen: count blinking lights, sort colored orbs, and hold a steady cursor on a moving target.
- **Variation:** One mini-game occasionally pauses, tempting the player to focus there, while the others accelerate.
- **Twist:** Letting one fail does not end the game—it changes the character’s response in the subsequent dialogue (snapping vs. withdrawing).

## Paranoia / Distrust
- **Mechanic:** Dialogue choices appear with identical visual treatment. One choice is randomly designated as "wrong" after selection, forcing the player to second-guess pattern recognition.
- **Variation:** UI elements occasionally flicker or duplicate, making it unclear if an input registered.
- **Twist:** Sometimes the "wrong" choice produces the better narrative outcome, training the player to distrust the system’s feedback.

## Euphoria / Mania
- **Mechanic:** Inputs produce exaggerated, cascading effects. One button press triggers multiple animations, sounds, and point showers.
- **Variation:** The character cannot stop moving; the player must steer continuous momentum through tight spaces.
- **Twist:** Higher scores correlate with riskier behavior. The optimal path is reckless.

## Shame / Guilt
- **Mechanic:** Reconstruct a scene from scattered pieces, but one piece is clearly labeled as "your fault." It does not fit cleanly with the others.
- **Variation:** Other pieces shift away from the fault piece, requiring the player to force the connection.
- **Twist:** Completing the puzzle with the fault piece integrated changes the color palette to something warmer. Avoiding it leaves the scene cold and incomplete.

## Dissociation
- **Mechanic:** Control separates into two layers. Player inputs affect the world 2 seconds later. The character and environment drift out of sync.
- **Variation:** The screen splits. One side shows reality, the other shows a filtered or distorted version. Player must navigate using both.
- **Twist:** Actions in the distorted side affect the real side unpredictably.
